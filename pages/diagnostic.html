Diagnostic système
Tests de santé en un clic depuis votre session connectée. Aucun secret n'est exposé.

1. Session (front)

En attente

Affiche l'utilisateur courant via le client Supabase front.


  


  

    

      
2. Token → Supabase (direct)

      En attente
    

    
Valide l'access_token de la session directement contre l'API Supabase.

    
    

  


  

    

      
3. Token → notre API

      En attente
    

    
Vérifie que nos endpoints serveur valident ce même token (GET /api/channel-property).

    
    

  


  

    

      
4. Webhook register

      En attente
    

    
Enregistre le webhook global chez le gestionnaire de canaux (POST /api/channel-webhook).

    
    

  


  

    

      
5. API canaux (prod)

      En attente
    

    
Teste la connexion live au gestionnaire de canaux (CHANNEL_BASE_URL + clé prod).

    
    

  


  

    

      
6. Import des messages d'un bien

      En attente
    

    
Importe l'historique des messages du bien (POST /api/channel-import-messages).

    
    
    

  


  

    

      
7. Lire les canaux d'un bien

      En attente
    

    
Liste les canaux (Airbnb, Booking…) du bien puis affiche l'objet complet de chacun : mappings room/rate, is_active, tous les attributs. Appel serveur (GET only), valeurs sensibles masquées.

    
    
    

  


  

    

      
8. Register webhook events (canal)

      En attente
    

    
Enregistre le 2e webhook séparé pour les events canal (new_channel;updated_channel;activate_channel) → POST /api/channel-events. Le webhook booking/message reste inchangé.

    
    

  


  

    

      
9. Group ID du bien

      En attente
    

    
GET /groups → identifie le group_id propriétaire du bien (requis pour créer un canal). Lecture seule. Le champ ci-dessous sert aussi aux tests 10 et 11.

    
    
    

  


  

    

      
10. Canaux du bien (état)

      En attente
    

    
GET /channels?filter[property_id] → voir si l'OAuth crée un canal nu (réserve 1) et son is_active. Utilise le champ du test 9.

    
    

  


  

    

      
11. mapping_details (annonces Airbnb)

      En attente
    

    
POST /channels/mapping_details du canal → rooms[]/rates[] Airbnb + codes entiers room_type_code/rate_plan_code (réserve 2 + point 4). Utilise le champ du test 9.

    
    

  


  

    

      
12. Lister les annonces Airbnb

      En attente
    

    
POST /channels/mapping_details → listing_id_dictionary (les annonces du compte, pour l'écran de choix). Lecture. Utilise le champ du test 9.

    
    

  


  

    

      
13. Mapper — DRY-RUN (n'envoie rien)

      En attente
    

    
Construit le payload map (group_id + rate_plan_id + settings clonés du canal réel, listing_id remplacé) et l'affiche sans l'envoyer. Utilise le champ du test 9.

    
    
    

  


  

    

      
14. Activer — DRY-RUN (n'envoie rien)

      En attente
    

    
Montre l'appel POST /channels/:id/activate sans l'envoyer. Renseigne un channel_id (ne PAS utiliser celui de Colomiers pour un vrai envoi).
